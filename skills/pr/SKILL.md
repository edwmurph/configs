# pr

Use this skill when creating a pull request.

## Goal
Write a PR description that is minimal, clear, and easy to scan.

## Instructions
1. Check whether the repository has a PR template.
2. If a template exists, follow its structure and required sections.
3. Keep all content concise and plain.
4. If no template exists, write:
   - A short opening sentence with context.
   - A bullet list of the key changes.
5. Avoid long narratives, repetition, or unnecessary detail.

## Output style
- Minimal and clear.
- Prefer bullets for changes.
- Include only details that help reviewers understand scope and impact.
